## 概述

本目录包含用于构建和运行 Odoo 19.0 的 Docker 镜像配置。该配置基于 Ubuntu Noble，包含完整的 Odoo 19.0 运行环境和必要的依赖。

### 主要特性

- ✅ 基于 Ubuntu Noble 基础镜像
- ✅ 预装 Odoo 19.0 完整环境
- ✅ 包含 PostgreSQL 客户端工具
- ✅ 自动等待数据库就绪机制
- ✅ 支持自定义插件挂载
- ✅ 数据持久化支持

## 核心文件详解

### 1. Dockerfile

#### 主要功能

1. **基础环境设置**
   - 使用 Ubuntu Noble 作为基础镜像
   - 设置语言环境为 `en_US.UTF-8`
   - 支持多架构（amd64, arm64, ppc64le）

2. **系统依赖安装**
   - 安装 Python 3 相关库（pip, setuptools 等）
   - 安装 Odoo 所需的 Python 包（num2words, phonenumbers, qrcode 等）
   - 安装 Node.js 和 lessc（用于 CSS 编译）
   - 安装 wkhtmltopdf（用于 PDF 生成）

3. **PostgreSQL 客户端**
   - 安装最新版本的 PostgreSQL 客户端
   - 用于数据库连接和操作

4. **Odoo 安装**
   - 从官方源下载并安装 Odoo 19.0
   - 版本：`19.0.20251208`
   - 使用 SHA 校验确保文件完整性

5. **配置和脚本**
   - 复制 `entrypoint.sh` 到容器根目录
   - 复制 `odoo.conf` 到 `/etc/odoo/`
   - 复制 `wait-for-psql.py` 到 `/usr/local/bin/`

6. **数据卷和端口**
   - 挂载 `/var/lib/odoo`（Odoo 数据目录）
   - 挂载 `/mnt/extra-addons`（自定义插件目录）
   - 暴露端口：8069（HTTP）、8071（HTTPS）、8072（长轮询）

7. **用户和入口**
   - 使用 `odoo` 用户运行（非 root，提高安全性）
   - 设置入口点为 `entrypoint.sh`

#### 关键配置

```dockerfile
ENV ODOO_VERSION 19.0
ARG ODOO_RELEASE=20251208
ARG ODOO_SHA=76c8b61b443676477eea546635aca37b8431dd9d
```

### 2. entrypoint.sh

#### 主要功能

1. **密码文件支持**

   ```bash
   if [ -v PASSWORD_FILE ]; then
       PASSWORD="$(< $PASSWORD_FILE)"
   fi
   ```

   - 支持从文件读取密码（提高安全性）

2. **环境变量处理**
   - 从环境变量读取数据库配置
   - 支持多种环境变量格式（兼容 Docker Compose 和 Kubernetes）
   - 默认值设置：
     - `HOST`: `db`（或 `DB_PORT_5432_TCP_ADDR`）
     - `PORT`: `5432`（或 `DB_PORT_5432_TCP_PORT`）
     - `USER`: `odoo`（或 `DB_ENV_POSTGRES_USER` 或 `POSTGRES_USER`）
     - `PASSWORD`: `odoo`（或 `DB_ENV_POSTGRES_PASSWORD` 或 `POSTGRES_PASSWORD`）

3. **配置文件检查**
   - `check_config()` 函数检查 `odoo.conf` 中是否已配置参数
   - 如果配置文件中已有参数，使用配置文件的值
   - 否则使用环境变量的值

4. **启动逻辑**
   - 如果命令是 `odoo` 或 `--`，执行 Odoo
   - 如果是 `scaffold`（创建模块骨架），直接执行，不等待数据库
   - 其他情况先等待数据库就绪，再执行命令

#### 执行流程

```
启动容器
  ↓
读取环境变量（HOST, PORT, USER, PASSWORD）
  ↓
检查 odoo.conf 配置文件
  ↓
合并配置（配置文件优先）
  ↓
等待数据库就绪（wait-for-psql.py）
  ↓
启动 Odoo 服务
```

### 3. wait-for-psql.py

#### 主要功能

- 等待 PostgreSQL 数据库服务就绪
- 通过 `psycopg2` 尝试连接数据库
- 支持超时设置（默认 5 秒，entrypoint.sh 中设置为 30 秒）
- 连接成功后退出，失败则报错

#### 使用场景

在 Docker Compose 中，虽然 `depends_on` 可以确保启动顺序，但无法确保数据库已完全就绪。此脚本确保数据库真正可用后再启动 Odoo。

## 环境准备

### 系统要求

- **操作系统**: Linux、macOS 或 Windows（WSL2）
- **Docker**: 版本 20.10 或更高
- **Docker Compose**: 版本 2.0 或更高（可选，但推荐）
- **磁盘空间**: 至少 5GB 可用空间
- **内存**: 建议 4GB 或更多

### 安装 Docker

#### Ubuntu/Debian

```bash
# 更新包索引
sudo apt-get update

# 安装必要的依赖
sudo apt-get install -y \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

# 添加 Docker 官方 GPG 密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 添加 Docker 仓库
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户添加到 docker 组（可选，避免每次使用 sudo）
sudo usermod -aG docker $USER
# 需要重新登录才能生效
```

#### macOS

```bash
# 使用 Homebrew 安装
brew install --cask docker

# 或者从官网下载 Docker Desktop
# https://www.docker.com/products/docker-desktop
```

#### Windows

1. 安装 WSL2（Windows Subsystem for Linux 2）
2. 下载并安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop)

### 验证安装

```bash
# 检查 Docker 版本
docker --version

# 检查 Docker Compose 版本
docker compose version

# 运行测试容器
docker run hello-world
```

## 构建 Docker 镜像

### 方法一：使用 Dockerfile 直接构建

```bash
# 进入 19.0 目录
cd docker/19.0

# 构建镜像
docker build -t odoo:19.0-custom .

# 查看构建的镜像
docker images | grep odoo
```

#### 构建参数说明

```bash
# 指定目标架构（可选）
docker build --build-arg TARGETARCH=amd64 -t odoo:19.0-custom .

# 指定 Odoo 版本（可选，默认使用 Dockerfile 中的版本）
docker build --build-arg ODOO_RELEASE=20251208 -t odoo:19.0-custom .
```

### 方法二：使用 Docker Compose（推荐）

如果项目根目录有 `docker-compose.yml`，可以直接使用：

```bash
# 在 docker 目录下
cd docker

# 构建并启动所有服务
docker compose up -d --build

# 只构建，不启动
docker compose build
```

### 构建过程说明

1. **下载基础镜像**（Ubuntu Noble）
2. **安装系统依赖**（约 5-10 分钟）
3. **安装 PostgreSQL 客户端**（约 1-2 分钟）
4. **安装 wkhtmltopdf**（约 1 分钟）
5. **安装 Odoo**（约 2-3 分钟）
6. **复制配置文件和脚本**
7. **设置权限和用户**

**总耗时**: 约 10-20 分钟（取决于网络速度）

## 运行容器

### 方法一：使用 Docker 命令

#### 基本运行

```bash
# 运行容器（需要先启动 PostgreSQL）
docker run -d \
  --name odoo \
  -p 8069:8069 \
  -e HOST=db \
  -e PORT=5432 \
  -e USER=odoo \
  -e PASSWORD=odoo \
  -v odoo-data:/var/lib/odoo \
  --link db:db \
  odoo:19.0-custom
```

#### 挂载自定义插件

```bash
docker run -d \
  --name odoo \
  -p 8069:8069 \
  -e HOST=db \
  -e PORT=5432 \
  -e USER=odoo \
  -e PASSWORD=odoo \
  -v odoo-data:/var/lib/odoo \
  -v /path/to/your/addons:/mnt/extra-addons \
  --link db:db \
  odoo:19.0-custom
```

#### 使用自定义配置文件

```bash
docker run -d \
  --name odoo \
  -p 8069:8069 \
  -v /path/to/odoo.conf:/etc/odoo/odoo.conf \
  -v odoo-data:/var/lib/odoo \
  --link db:db \
  odoo:19.0-custom
```

### 方法二：使用 Docker Compose（推荐）

#### 创建 docker-compose.yml

```yaml
version: "3.8"

services:
  db:
    image: postgres:16
    environment:
      - POSTGRES_DB=postgres
      - POSTGRES_USER=odoo
      - POSTGRES_PASSWORD=odoo
      - PGDATA=/var/lib/postgresql/data/pgdata
    volumes:
      - odoo-db-data:/var/lib/postgresql/data/pgdata
    restart: always

  web:
    build:
      context: ./19.0
      dockerfile: Dockerfile
    depends_on:
      - db
    ports:
      - "8069:8069"
    environment:
      - HOST=db
      - PORT=5432
      - USER=odoo
      - PASSWORD=odoo
    volumes:
      - odoo-web-data:/var/lib/odoo
      - ./addons:/mnt/extra-addons # 挂载自定义插件
      - ./config/odoo.conf:/etc/odoo/odoo.conf # 挂载配置文件
    restart: unless-stopped

volumes:
  odoo-web-data:
  odoo-db-data:
```

#### 启动服务

```bash
# 启动所有服务（后台运行）
docker compose up -d

# 启动并查看日志
docker compose up

# 只启动特定服务
docker compose up -d web
```

### 访问 Odoo

启动成功后，访问：

- **Odoo Web 界面**: http://localhost:8069
- **数据库管理**: 通过 pgAdmin 或其他工具连接到 PostgreSQL

## 配置说明

### Odoo 配置文件 (odoo.conf)

配置文件位置：`/etc/odoo/odoo.conf`

#### 关键配置项

```ini
[options]
# 插件路径（包含系统插件和自定义插件）
addons_path = /usr/lib/python3/dist-packages/odoo/addons,/mnt/extra-addons

# 数据目录（文件存储）
data_dir = /var/lib/odoo

# 服务器级模块
server_wide_modules = base,web

# 代理模式（生产环境建议开启）
proxy_mode = False

# 管理员密码（用于数据库管理）
admin_passwd = admin

# 数据库配置（可通过环境变量覆盖）
db_host = db
db_port = 5432
db_user = odoo
db_password = odoo

# 数据库名称（False 表示显示数据库列表）
db_name = False

# 数据库过滤（.* 表示所有数据库）
dbfilter = .*

# 日志级别
log_level = info

# 工作进程数（0 表示自动，建议设置为 CPU 核心数）
workers = 0

# XML-RPC 端口
xmlrpc_port = 8069

# 长轮询端口
longpolling_port = 8072
```

### 配置文件优先级

1. **环境变量**（最高优先级，在 entrypoint.sh 中处理）
2. **odoo.conf 配置文件**
3. **Odoo 默认值**

## 环境变量

### 数据库连接变量

| 变量名          | 说明         | 默认值 | 示例                           |
| --------------- | ------------ | ------ | ------------------------------ |
| `HOST`          | 数据库主机   | `db`   | `localhost` 或 `192.168.1.100` |
| `PORT`          | 数据库端口   | `5432` | `5432`                         |
| `USER`          | 数据库用户   | `odoo` | `odoo`                         |
| `PASSWORD`      | 数据库密码   | `odoo` | `your_password`                |
| `PASSWORD_FILE` | 密码文件路径 | -      | `/run/secrets/db_password`     |

### 兼容性变量

为了兼容不同的部署方式，entrypoint.sh 支持以下变量：

- `DB_PORT_5432_TCP_ADDR`（Docker link 方式）
- `DB_PORT_5432_TCP_PORT`（Docker link 方式）
- `DB_ENV_POSTGRES_USER`（Docker Compose 方式）
- `DB_ENV_POSTGRES_PASSWORD`（Docker Compose 方式）
- `POSTGRES_USER`（标准 PostgreSQL 变量）
- `POSTGRES_PASSWORD`（标准 PostgreSQL 变量）

### 使用示例

#### Docker Compose

```yaml
services:
  web:
    environment:
      - HOST=db
      - PORT=5432
      - USER=odoo
      - PASSWORD=my_secure_password
      # 或使用密码文件（更安全）
      - PASSWORD_FILE=/run/secrets/db_password
    secrets:
      - db_password
```

#### Docker 命令

```bash
docker run -d \
  -e HOST=db \
  -e PORT=5432 \
  -e USER=odoo \
  -e PASSWORD=my_secure_password \
  odoo:19.0-custom
```

#### Kubernetes

```yaml
env:
  - name: HOST
    value: "postgres-service"
  - name: PORT
    value: "5432"
  - name: USER
    valueFrom:
      secretKeyRef:
        name: odoo-secrets
        key: db-user
  - name: PASSWORD
    valueFrom:
      secretKeyRef:
        name: odoo-secrets
        key: db-password
```

## 故障排除

### 问题 1: 容器无法启动

#### 症状

```bash
docker ps -a
# 容器状态显示 Exited (1)
```

#### 排查步骤

1. **查看日志**

   ```bash
   docker logs odoo
   ```

2. **检查数据库连接**
   - 确认数据库容器正在运行：`docker ps | grep db`
   - 检查数据库配置是否正确
   - 验证网络连接：`docker network ls`

3. **检查端口占用**

   ```bash
   # Linux/macOS
   netstat -an | grep 8069
   # 或
   lsof -i :8069

   # Windows
   netstat -an | findstr 8069
   ```

4. **检查权限**
   ```bash
   # 检查数据卷权限
   docker exec -it odoo ls -la /var/lib/odoo
   ```

#### 解决方案

- 如果端口被占用，修改端口映射：`-p 8070:8069`
- 如果数据库连接失败，检查环境变量和配置文件
- 如果权限问题，检查挂载的目录权限

### 问题 2: 数据库连接超时

#### 症状

```
Database connection failure: could not connect to server
```

#### 排查步骤

1. **检查数据库服务**

   ```bash
   docker ps | grep db
   docker logs db
   ```

2. **检查网络**

   ```bash
   # 测试数据库连接
   docker exec odoo psql -h db -U odoo -d postgres -c "SELECT 1;"
   ```

3. **检查环境变量**
   ```bash
   docker exec odoo env | grep -E "HOST|PORT|USER|PASSWORD"
   ```

#### 解决方案

- 确保数据库容器已启动：`docker compose up -d db`
- 检查 `HOST` 环境变量是否正确（应该是服务名 `db`，不是 `localhost`）
- 增加等待超时时间（修改 `entrypoint.sh` 中的 `--timeout=30`）

### 问题 3: 无法访问 Odoo Web 界面

#### 症状

- 浏览器无法连接到 http://localhost:8069
- 连接超时

#### 排查步骤

1. **检查容器状态**

   ```bash
   docker ps | grep odoo
   ```

2. **检查端口映射**

   ```bash
   docker port odoo
   ```

3. **检查防火墙**

   ```bash
   # Linux
   sudo ufw status
   sudo iptables -L
   ```

4. **查看 Odoo 日志**
   ```bash
   docker logs odoo | tail -50
   ```

#### 解决方案

- 确认端口映射正确：`-p 8069:8069`
- 检查防火墙规则，允许 8069 端口
- 如果是 Docker Desktop，检查端口转发设置

### 问题 4: 插件未加载

#### 症状

- 自定义插件在 Odoo 中不可见
- 插件路径配置错误

#### 排查步骤

1. **检查插件路径**

   ```bash
   docker exec odoo odoo -d my_database --addons-path=/usr/lib/python3/dist-packages/odoo/addons,/mnt/extra-addons --stop-after-init
   ```

2. **检查挂载**

   ```bash
   docker exec odoo ls -la /mnt/extra-addons
   ```

3. **检查插件目录结构**
   ```bash
   # 插件目录应包含 __manifest__.py
   ls -la /path/to/your/addons/my_module/
   ```

#### 解决方案

- 确保插件目录已正确挂载：`-v /path/to/addons:/mnt/extra-addons`
- 在 `odoo.conf` 中添加插件路径：`addons_path = /usr/lib/python3/dist-packages/odoo/addons,/mnt/extra-addons`
- 检查插件目录权限

### 问题 5: 内存不足

#### 症状

- 容器频繁重启
- Odoo 运行缓慢
- 日志显示内存错误

#### 排查步骤

1. **检查内存使用**

   ```bash
   docker stats odoo
   ```

2. **检查系统内存**
   ```bash
   free -h
   ```

#### 解决方案

- 增加 Docker 内存限制（Docker Desktop 设置）
- 减少 Odoo 工作进程数（在 `odoo.conf` 中设置 `workers = 2`）
- 增加系统内存或使用交换空间

### 问题 6: 数据丢失

#### 症状

- 重启容器后数据消失
- 文件上传后找不到

#### 排查步骤

1. **检查数据卷**

   ```bash
   docker volume ls
   docker volume inspect odoo-web-data
   ```

2. **检查挂载**
   ```bash
   docker inspect odoo | grep -A 10 Mounts
   ```

#### 解决方案

- 确保使用数据卷或绑定挂载：`-v odoo-data:/var/lib/odoo`
- 不要使用临时存储
- 定期备份数据卷

## 最佳实践

### 1. 安全性

#### 使用密码文件

```bash
# 创建密码文件
echo "my_secure_password" > db_password.txt
chmod 600 db_password.txt

# 使用密码文件
docker run -d \
  -e PASSWORD_FILE=/run/secrets/db_password \
  -v $(pwd)/db_password.txt:/run/secrets/db_password:ro \
  odoo:19.0-custom
```

#### 修改默认密码

```bash
# 修改 admin_passwd
docker exec -it odoo sed -i 's/admin_passwd = admin/admin_passwd = your_secure_password/' /etc/odoo/odoo.conf
docker restart odoo
```

#### 使用非 root 用户

容器默认使用 `odoo` 用户运行，这是安全的。不要以 root 用户运行。

### 2. 性能优化

#### 配置工作进程

```ini
# odoo.conf
workers = 4  # 设置为 CPU 核心数
```

#### 启用代理模式

```ini
# odoo.conf（生产环境）
proxy_mode = True
```

#### 数据库连接池

```ini
# odoo.conf
db_maxconn = 64
```

### 3. 数据管理

#### 定期备份

```bash
#!/bin/bash
# backup.sh
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backups"
mkdir -p $BACKUP_DIR

# 备份数据库
docker exec db pg_dump -U odoo postgres | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 备份文件存储
docker run --rm -v odoo-web-data:/data -v $BACKUP_DIR:/backup \
  ubuntu tar czf /backup/filestore_$DATE.tar.gz /data

echo "Backup completed: $DATE"
```

#### 设置备份计划（cron）

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 2 点备份
0 2 * * * /path/to/backup.sh
```
